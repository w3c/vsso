IRI: https://github.com/w3c/vsso#ActuatableVehicleProperty

_ActuatableVehicleProperty_ - Follows the definition of an [actuator](https://covesa.github.io/vehicle_signal_specification/rule_set/data_entry/sensor_actuator/) in VSS. It is a [DynamicVehicleProperty](#https://github.com/w3c/vsso#DynamicVehicleProperty), which can be changed. Refers to [sosa:ActuableProperty](https://www.w3.org/TR/vocab-ssn/#SOSAActuatableProperty)
has super-classes
DynamicVehicleProperty c

DynamicVehiclePropertyc back to ToC or Class ToC

IRI: https://github.com/w3c/vsso#DynamicVehicleProperty

_DynamicVehicleProperty_ - In contrast to a [StaticVehicleProperty](#https://github.com/w3c/vsso#StaticVehicleProperty), this property is expected to change frequently, even within a vehicle lifecycle. Examples are the vehicle `speed`, `location`, etc.
has super-classes
VehicleProperty c
has sub-classes
ActuatableVehicleProperty c, ObservableVehicleProperty c
is in range of
hasDynamicVehicleProperty op

ObservableVehiclePropertyc back to ToC or Class ToC

IRI: https://github.com/w3c/vsso#ObservableVehicleProperty

_ObservableVehicleProperty_ - Follows the definition of an [actuator](https://covesa.github.io/vehicle_signal_specification/rule_set/data_entry/sensor_actuator/) in VSS. It is a [DynamicVehicleProperty](#https://github.com/w3c/vsso#DynamicVehicleProperty), which can be observed but _not_ changed. Refers to [sosa:ObservableProperty](https://www.w3.org/TR/vocab-ssn/#SOSAObservableProperty)
has super-classes
DynamicVehicleProperty c

PositionInVehiclec back to ToC or Class ToC

IRI: https://github.com/w3c/vsso#PositionInVehicle

_PositionInVehicle_ - Frequently VehicleProperties and VehicleComponents will appear in several different positions in the vehicle. Typicle examples are windows, seats, etc. This concepts shall be used to define the different position of the vehicle. They can be named with the positionName property.
is in domain of
positionName dp
is in range of
postionedAt op

StaticVehiclePropertyc back to ToC or Class ToC

IRI: https://github.com/w3c/vsso#StaticVehicleProperty

_StaticVehicleProperty_ - Follows the definition of an [attribute](https://covesa.github.io/vehicle_signal_specification/rule_set/data_entry/attributes/) in VSS. `StaticVehicleProperty` is meant to stay stable at least over the duration of a vehicle lifecycle. If you expect more frequent updates, please consider [DynamicVehicleProperty](#https://github.com/w3c/vsso#DynamicVehicleProperty)
has super-classes
VehicleProperty c
is in range of
hasStaticVehicleProperty op

VehicleComponentc back to ToC or Class ToC

IRI: https://github.com/w3c/vsso#VehicleComponent

_VehicleComponent_ - First of all a sorting element for [VehicleProperties](#https://github.com/w3c/vsso#VehicleProperty), following on the definition of a [branch](https://covesa.github.io/vehicle_signal_specification/rule_set/branches/). It is encouraged to use the concept for actual parts modeled as `VehicleComponent.
is in domain of
partOfVehicle op, partOfVehicleComponent op
is in range of
belongsToVehicleComponent op, partOfVehicleComponent op

VehiclePropertyc back to ToC or Class ToC

IRI: https://github.com/w3c/vsso#VehicleProperty

_VehicleProperty_ - Central concept for defining a vehicle, either through a VehicleComponent structure or connected directly to the vehicle. Every VehicleProperty shall be instantiated with a [vehiclePropertyValue](#https://github.com/w3c/vsso#vehiclePropertyValue). It is good practice to set the time when the value was updated through [valueUpdatedAt](#https://github.com/w3c/vsso#valueUpdatedAt) The VehicleProperty follows the concept of a [ssn:Property](https://www.w3.org/TR/vocab-ssn/#SSNProperty)
has sub-classes
DynamicVehicleProperty c, StaticVehicleProperty c
is in domain of
belongsToVehicleComponent op, propertyValueUpdatedAt dp, vehiclePropertyValue dp
